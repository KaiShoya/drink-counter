# [T-0004] ナビバーのテーマボタンが SP / PC 両領域に二重レンダリングされている

**ID**: T-0004  
**作成日**: 2026-05-06  
**優先度**: Medium  
**種別**: bug  
**ステータス**: Backlog  
**担当バージョン**: v1.20  
**関連 Issue**: -

---

## Goal

`custom.vue` で `DomainHeaderAtomsThemeButton` が `navbar-brand`（モバイル用）と `navbar-end`（デスクトップ用）の両方に配置されており、DOM 上に同コンポーネントが二重に存在する問題を解消する。

## Scope（対象ファイル・範囲）

- `app/layouts/custom.vue`

## Out of Scope

- テーマ切り替え機能自体のロジック変更

## Plan

1. 現状のレンダリング挙動を SP / PC で確認し、どちらがどの条件で表示されているか整理
2. Bulma の `is-hidden-touch` / `is-hidden-desktop` クラスを使い、各領域の表示を排他制御する
3. `v-if="!active"` による制御と組み合わせて重複を排除
4. ユーザーアイコンのダミー `<div>` も合わせて整理する

## Verification

- [ ] SP（touch）でテーマボタンが 1 箇所のみ表示される
- [ ] PC（desktop）でテーマボタンが 1 箇所のみ表示される
- [ ] ハンバーガーメニュー開閉時にテーマボタンが正しく表示・非表示になる
- [ ] DOM インスペクターで `DomainHeaderAtomsThemeButton` が同時に 2 つ存在しないことを確認

## Notes

- 現在 `navbar-burger-left` クラスの挙動コメントに「ダミーのdivを追加した」とあり、burger 整列に根本的な問題がある可能性あり
- 修正前後でモバイル / デスクトップのナビ表示を目視確認すること
