## Goal

現在の `updateDrinksSort` は `drinks` 全件の配列インデックスを `sort` 値として上書きするため、ラベル詳細画面でラベルフィルタ済みサブリストのみを並び替えると他ラベルの飲み物の `sort` 値との整合が取れなくなる。ラベル内並び替えと全件並び替えが共存できるよう、ソートのスコープ設計を変更する。

## Scope（対象ファイル・範囲）

- `app/stores/data/drinks/actions.ts`（`updateDrinksSort` の変更）
- `app/repositories/drinksRepository.ts`（`updateSort` の引数変更が生じる場合）
- `app/stores/pages/drinks/index/actions.ts`（`save` のアクション呼び出し側）

## Out of Scope

- UI の変更（T-0020 で対応）
- ラベル詳細画面のストア追加（T-0020 で対応）

## Plan

1. `updateDrinksSort` を全件更新から**指定した drinks のみ**を更新する形にリファクタリングする（引数: `DrinkRow[]`）
2. 内部で `payload` を生成し `$drinksRepository.updateSort` へ渡す（既存シグネチャに変更なし）
3. `sort` 値の採番戦略を決定する
   - 案A: **全件の中での絶対インデックス** を維持（ラベルをまたいだ表示順序を保持）
   - 案B: **ラベル内の相対インデックス** のみ（ラベル横断の順序は未定義）
   - → **案A を採用**（drinks/index.vue の全件表示でも order by sort が機能するため）
   - 具体的には: 更新対象外の飲み物の sort を変えず、対象飲み物の sort 値を「更新対象外の sort 値の間に収まる連番」で採番する
4. `usePageDrinksStore` の `save` をリファクタリング後の関数呼び出しに変更する
5. 既存の `drinks/index.vue` の動作が変わらないことをテストで確認する

## Verification

- [ ] `drinks/index.vue` での全件並び替え保存が従来通り動作する
- [ ] ラベルフィルタ済みサブリストで `updateDrinksSort([filteredDrinks])` を呼んだとき、他ラベルの飲み物の `sort` 値が変更されない
- [ ] Supabase の `drinks` テーブルで `order('sort,id')` したとき、全件が意図した順番で取得される

## Notes

- `app/repositories/drinksRepository.ts` の `fetchAll` は `order('sort,id')` でソートして全件取得している
- 案A の採番例: 対象の飲み物を配列順で sort=0,1,2… とするのではなく、「対象飲み物を除いた他の飲み物の最大 sort 値 + 1」から連番を振ることで衝突を防ぐ方法も検討
- T-0020 の実装前にこのタスクを完了させること（依存関係）
