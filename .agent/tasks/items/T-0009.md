# [T-0009] 設定ページの数値入力に min 属性がなく負数入力が可能

**ID**: T-0009  
**作成日**: 2026-05-06  
**優先度**: Low  
**種別**: enhancement  
**ステータス**: Backlog  
**担当バージョン**: -  
**関連 Issue**: -

---

## Goal

過飲閾値・日付切替時刻の `input[type="number"]` に `min` 属性がなく、負数や不正な値が入力できる。UI 側で最低限の入力制約を追加する。

## Scope（対象ファイル・範囲）

- `app/pages/(protected)/settings.vue`

## Out of Scope

- Store / Repository 側のバリデーション追加（既存ロジックに委ねる）
- `max` 属性の設定（要件が決まってから対応）

## Plan

1. 過飲閾値の `<input type="number">` に `min="1"` を追加（0杯以下は意味がないため）
2. 日付切替時刻の `<input type="number">` に `min="0"` `max="23"` を追加

## Verification

- [ ] 過飲閾値に 0 以下を入力しようとすると、ブラウザのバリデーションが機能する
- [ ] 日付切替時刻に 0〜23 以外の値を入力しようとすると、ブラウザのバリデーションが機能する
- [ ] 正常な値での保存フローに影響がない

## Notes

- `min` はあくまで UI ヒント。Store 側のバリデーションと整合が取れているか確認すること
- T-0007（設定フォームのリファクタリング）と合わせて対応するとまとめやすい
