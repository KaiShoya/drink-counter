## Goal

飲み物の編集・新規フォームでクライアントサイドバリデーションを追加し、サーバーエラーに頼らずにユーザーへインラインエラーを表示する。

## Scope（対象ファイル・範囲）

- `app/components/domain/drink/atoms/Edit.vue`

## Out of Scope

- ラベルフォームのバリデーション（別途対応）
- サーバーサイドバリデーションの変更

## Plan

1. バリデーションルールを定義する:
   - `name`: 必須・1文字以上
   - `color`: 必須・`#RRGGBB` または `#RGB` 形式の正規表現チェック
   - `amount`: 必須・正の整数（1以上）
2. `computed` で `errors` オブジェクト（各フィールドのエラーメッセージ）を算出
3. Bulma の `is-danger` クラスと `<p class="help is-danger">` でインラインエラーを表示
4. `saveFunction()` 呼び出し前にバリデーションを実行し、エラーがあれば早期 return
5. i18n キーでエラーメッセージを管理する（`locales.ts` に追加）

## Verification

- [ ] name が空の場合、保存ボタン押下でエラーメッセージが表示される
- [ ] color が無効なHEX文字列の場合、エラーが表示される
- [ ] amount が 0 以下または非数値の場合、エラーが表示される
- [ ] すべてのバリデーションを通過した場合のみ saveFunction が呼ばれる
- [ ] バリデーションエラーのテキストが ja/en で表示される

## Notes

- カラーピッカーからの入力は常に有効なHEXなので、テキスト入力のみバリデーション対象
- HEX バリデーション正規表現: `/^#([0-9A-Fa-f]{3}|[0-9A-Fa-f]{6})$/`
