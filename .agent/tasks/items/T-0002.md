# [T-0002] BaseAtomsButton に disabled prop がなく無効化できない

**ID**: T-0002  
**作成日**: 2026-05-06  
**優先度**: High  
**種別**: bug  
**ステータス**: Backlog  
**担当バージョン**: v1.20  
**関連 Issue**: -

---

## Goal

飲み物が未設定のラベル行で ± ボタンが視覚的・機能的に操作可能に見える問題を修正する。`BaseAtomsButton` に `disabled` prop を追加し、`MinusButton` / `PlusButton` 経由で伝播させる。

## Scope（対象ファイル・範囲）

- `app/components/base/atoms/Button.vue`
- `app/components/domain/counter/atoms/MinusButton.vue`
- `app/components/domain/counter/atoms/PlusButton.vue`
- `app/components/domain/counter/molecules/Row.vue`（`disabled` prop の受け渡し）

## Out of Scope

- DatePicker の ± ボタンへの適用（別コンポーネントのため別途判断）

## Plan

1. `Button.vue` に `disabled?: boolean` prop を追加し、`<button :disabled="disabled">` に反映
2. `MinusButton.vue` / `PlusButton.vue` に `disabled?: boolean` prop を追加して `BaseAtomsButton` へ渡す
3. `Row.vue` の `<DomainCounterAtomsMinusButton :disabled="!label.currentDrink" />` と `<DomainCounterAtomsPlusButton :disabled="!label.currentDrink" />` に prop を追加（`ColumnDisabled` 分岐内）

## Verification

- [ ] 飲み物未設定のラベル行で ± ボタンが `disabled` 状態になり、クリック不可になる
- [ ] 飲み物設定済みのラベル行では ± ボタンが正常に動作する
- [ ] ボタンの `aria-label` は既存のまま維持される

## Notes

- `button:disabled` のスタイルは Bulma が提供しているため、追加 CSS は不要なはず
- `ColumnDisabled` テンプレートでは既に `disabled` prop を渡そうとしているが、`BaseAtomsButton` 側に prop がないため効いていない
