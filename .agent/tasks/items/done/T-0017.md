## Goal

飲み物・ラベルの編集フォームでキャンセルや別ページへの遷移を試みた際、未保存の変更がある場合に確認ダイアログを表示してデータ消失を防ぐ。

## Scope（対象ファイル・範囲）

- `app/pages/(protected)/drinks/[id].vue`
- `app/pages/(protected)/drinks/new.vue`
- `app/pages/(protected)/labels/[id].vue`
- `app/pages/(protected)/labels/new.vue`

## Out of Scope

- ドラッグ並び替えの離脱ガード（T-0015 で対応）

## Plan

1. 各ページで `isDirty` を算出する（初期値と現在のフォーム値の比較）
2. `onBeforeRouteLeave` フックで `isDirty` が `true` の場合に `window.confirm` を表示
3. ユーザーが「離脱する」を選択した場合のみナビゲーションを続行
4. キャンセルボタン（`<NuxtLink :to="'/drinks'">`）を `<button @click="handleCancel">` に変更し、同様の確認を入れる
5. i18n キーで確認メッセージを管理する

## Verification

- [x] フォームを変更後にキャンセルボタンを押すと確認ダイアログが出る
- [x] 確認ダイアログで「離脱する」を選ぶと遷移できる
- [x] 「キャンセル」を選ぶとフォームに留まる
- [x] 変更なしでキャンセルを押した場合は確認なしで遷移する
- [x] ブラウザのページ戻るボタンでも同様に確認が出る

## Notes

- Vue Router の `onBeforeRouteLeave` は Nuxt 4 でも使用可能
- `window.confirm` はシンプルだが、プロジェクトの共通モーダルコンポーネント (`CommonModalMoleculesDanger`) を使う方がスタイル統一の観点から望ましい
