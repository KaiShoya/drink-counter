# [T-0003] DatePicker のボタンと input の高さ不一致

**ID**: T-0003  
**作成日**: 2026-05-06  
**優先度**: Medium  
**種別**: bug  
**ステータス**: done  
**担当バージョン**: v1.20  
**関連 Issue**: -

---

## Goal

日付ピッカー行で前後ボタン（38px）と `input.is-large`（62px）の高さが揃っていない問題を解消し、ピッカー行のレイアウトを統一する。

## Scope（対象ファイル・範囲）

- `app/components/domain/picker/molecules/DatePicker.vue`
- `app/assets/scss/index.scss`（必要に応じて `.button.is-button-large` の高さ修正）

## Out of Scope

- MonthPicker / YearPicker の対応（同様の問題があれば別タスクで対応）

## Plan

1. 方針を決定:
   - A. `input` の高さを 38px（ボタンに合わせる）→ `is-large` をやめて通常サイズにする
   - B. ボタンの高さを 62px に合わせる（`.button.is-button-large` を更新）
2. 決定した方針で SCSS / テンプレートを修正
3. SP・PC 両方で目視確認

## Verification

- [x] DatePicker の前後ボタンと input が同じ高さで揃っている（SP / PC 両方）
- [x] 既存のボタンサイズを使用している他コンポーネントへの影響なし（`BaseAtomsButton` の `.button.is-button-large` は変更不要）

## Work Log

- 2026-05-07: picker `MinusButton.vue` / `PlusButton.vue` から `is-large` を除去し Bulma の height 計算との競合を解消
- scoped スタイルに `font-size: 1.25rem` を明示しアイコンサイズを維持
- `TodayButton.vue` は既に `is-large` なし・ scoped `height: 62px` のみで問題なし

## Notes

- `input.is-large.radius-unset { height: 62px }` は `index.scss` でカスタム定義されている
- ボタン高さを変更する場合は `.button.is-button-large` が使われている箇所を全確認する
