## Goal

飲み物・ラベルの編集/新規フォームで、保存処理中にボタンを `is-loading` + `disabled` にして二重送信を防ぐ。

## Scope（対象ファイル・範囲）

- `app/components/domain/drink/atoms/Edit.vue`
- `app/components/domain/label/atoms/Edit.vue`（存在確認後に対応）
- `app/stores/pages/drinks/edit/actions.ts`
- `app/stores/pages/drinks/new/actions.ts`
- `app/stores/pages/labels/edit/actions.ts`
- `app/stores/pages/labels/new/actions.ts`

## Out of Scope

- トップページのカウンター増減ボタン（レスポンスが非常に速いため対応優先度低）

## Plan

1. 各ストアの state に `isSaving: boolean` を追加（初期値 `false`）
2. `updateDrinkById` / `create` / `update` の前後で `isSaving` を `true` / `false` にセット（try/finally で確実に戻す）
3. `DomainDrinkAtomsEdit` に `isSaving` prop を追加し、保存ボタンに `:class="{ 'is-loading': isSaving }"` と `:disabled="isSaving"` を適用
4. 同様の対応をラベル編集コンポーネントにも行う

## Verification

- [x] 保存ボタンを押した直後からレスポンス返却までボタンが `is-loading` 状態になる
- [x] `is-loading` 中に保存ボタンを追加クリックしても API が二重送信されない
- [x] エラー時にも `isSaving` が `false` に戻る（try/finally で保証）

## Notes

- Bulma の `is-loading` クラスはボタン内にスピナーを表示する
- try/finally パターンで確実に状態をリセットすること
