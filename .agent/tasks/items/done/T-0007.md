## Goal

`settings.vue` のフォームが `<table>` で実装されており、`<label>` と `<input>` の関連付けがなくスクリーンリーダー非対応。フォームとして適切なセマンティクスに変更する。

## Scope（対象ファイル・範囲）

- `app/pages/(protected)/settings.vue`

## Out of Scope

- 設定項目の追加・削除
- 設定ロジック（store）の変更

## Plan

1. `<table>` ベースのレイアウトを `<div>` + Bulma の `field` / `control` クラスに置き換える
2. 各入力要素に `id` を付与し、対応する `<label for="...">` を追加する
3. 既存のスタイル（`is-hoverable is-striped` 等）を代替 UI で維持または調整する

## Verification

- [x] 各入力フィールドに `<label>` が関連付けられている（`for` / `id`）
- [x] スクリーンリーダーで各フィールドが正しくアナウンスされる（Bulma field / label の正規セマンティクスで対応）
- [x] 既存の設定保存フローが正常に動作する（store 連携は未変更）

## Work Log

- 2026-05-07: `settings.vue` の `<table>` を Bulma `field is-horizontal` / `field-body` / `control` 構造に置き換え
- `threshold` / `timezone` / `switching-timing` の各 input に `id` を付与し `<label for>` で関連付け
- 単位表示（杯 / 時）を `is-static` ボタン（`has-addons`）で隐永的に表示
- `<spec>` の `## Accessibility` セクションを完了済みに更新

## Notes

- i18n キーはそのまま維持する（`LOCALE_SETTINGS_*`）
- `<spec>` ブロックの `## Accessibility` セクションも同時に更新すること
