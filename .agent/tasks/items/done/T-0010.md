## Goal

飲み物の編集を別ページ遷移ではなくインラインのモーダルで完結させ、ラベル詳細（このラベルの飲み物）のコンテキストを維持したまま編集できるようにする。

## Scope（対象ファイル・範囲）

- `app/pages/(protected)/drinks/[id].vue`（ラベル詳細内の飲み物編集/追加をモーダル化）
- `app/pages/(protected)/drinks/item/[id].vue`（フォールバックルートとして維持）
- `app/pages/(protected)/drinks/item/new.vue`（フォールバックルートとして維持）
- `app/components/domain/drink/atoms/Edit.vue`（フォーム本体はそのまま再利用）
- `app/stores/pages/drinks/edit/`（initPage のルートパラム依存を解消）
- `app/stores/pages/drinks/new/`

## Out of Scope

- ラベル編集 (`/labels/:id`) の同様対応（別タスク化して構わない）
- Edit.vue のバリデーション追加（T-0014 で対応）

## Plan

1. `PageDrinkEditStore.initPage()` を route.params.id に依存しない形に変更し、drinkId を引数で受け取れるようにする
2. `drinks/[id].vue` にモーダル状態（新規/編集）を追加し、飲み物行の編集ボタンでモーダルを開く
3. 「このラベルに追加」導線を遷移リンクからモーダル起動に変更する
4. `DomainDrinkAtomsEdit` をモーダル内で再利用する
5. 保存/キャンセル後にモーダルを閉じ、一覧は store の更新で自動反映させる
6. `drinks/item/[id]` / `drinks/item/new` は直接アクセス時のフォールバックとして維持する

## Verification

- [x] ラベル詳細画面の編集アイコンをクリックするとモーダルが開き、フォームに既存データが入っている
- [x] 保存するとモーダルが閉じて一覧が更新される（store 更新反映）
- [x] キャンセルするとモーダルが閉じて一覧は変更されない
- [ ] モバイル幅でモーダルが正常に表示される（手動確認未実施）

## Validation

- `get_errors`（変更ファイル）: エラーなし
- `pnpm -s nuxi typecheck`: 失敗（既存エラー + 環境依存の `vue-tsc` 実行依存不足）。本タスク差分に起因するエラーは解消済み
