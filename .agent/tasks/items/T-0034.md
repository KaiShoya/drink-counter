## Goal

ネットワーク不安定時でも最低限の閲覧・記録が継続できるよう、オフラインキャッシュのMVPを導入する。

## Scope（対象ファイル・範囲）

- `app/repositories/**`
- `app/stores/**`
- `app/plugins/**`（必要に応じて）
- `.agent/specs/api-communication.md`（実装時更新）

## Out of Scope

- 完全同期基盤（競合解決を含む）
- バックグラウンド同期の高度実装

## Dependencies

- なし

## Plan

1. オフライン時の対象データ（読み取り/書き込み）をMVP定義する。
2. キャッシュ層を導入し、オンライン復帰時の反映フローを決める。
3. UI 上でオフライン状態を表示する。

## Verification

- [ ] オフライン時に直近データを表示できる
- [ ] オフライン時の操作可否が明確に示される
- [ ] オンライン復帰後に整合性が保たれる

## Next Action

Issue #292 の議論に合わせ、MVP対象を「閲覧のみ」か「記録まで含む」か先に確定する。

## Notes

- 仕様変更が大きいため `.agent/specs` 更新は必須。
