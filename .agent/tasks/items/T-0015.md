## Goal

飲み物・ラベルの一覧でドラッグ並び替え後に「並び順を保存」を押さずにページを離脱した場合、変更が消えることをユーザーに警告する（または自動保存する）。

## Scope（対象ファイル・範囲）

- `app/pages/(protected)/drinks/index.vue`
- `app/pages/(protected)/labels/index.vue`
- `app/stores/pages/drinks/index/`（unsaved フラグの管理）
- `app/stores/pages/labels/index/`（同上）

## Out of Scope

- ドラッグ以外（visible トグルなど）の変更への離脱ガード

## Plan

### オプション A: 離脱ガード（推奨）
1. ストアに `hasUnsavedSort: boolean` を追加
2. draggable の `@end` イベントで `hasUnsavedSort = true` にセット
3. 「並び順を保存」ボタン押下後に `hasUnsavedSort = false` にリセット
4. Nuxt の `onBeforeRouteLeave` で `hasUnsavedSort` が `true` の場合に確認ダイアログを表示する
5. Nuxt/Vue には組み込みの beforeunload フックがないため、`window.onbeforeunload` も併用する

### オプション B: 自動保存（debounce）
1. draggable の `@end` イベントで `save()` を debounce 300ms で呼ぶ
2. 「並び順を保存」ボタンは残し、手動保存も可能にする

## Verification

- [ ] 並び替え後に「並び順を保存」を押さずにナビゲーションすると確認が出る（オプション A）
- [ ] 保存後に離脱しても確認が出ない
- [ ] ページリロード時も警告が出る（`window.onbeforeunload`）

## Notes

- オプション A の方が意図が明確で UX も自然。まずこちらを実装する
- `onBeforeRouteLeave` は Vue Router のコンポーザブル
