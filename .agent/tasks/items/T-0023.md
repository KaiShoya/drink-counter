# [T-0023] 集計条件の設定連動を月次・年次画面で統一

## Goal

設定画面で変更したタイムゾーン・日付切替時刻を、月次/年次の集計取得条件へ反映し、表示値とユーザー設定の不整合を解消する。

## Scope（対象ファイル・範囲）

- `app/pages/(protected)/settings.vue`
- `app/pages/(protected)/data/monthly.vue`
- `app/pages/(protected)/data/annual.vue`
- `app/stores/pages/data/**`（必要に応じて）

## Out of Scope

- 集計ロジック自体のアルゴリズム変更
- 新規チャート種別の追加

## Dependencies

- なし

## Plan

1. 設定値（timezone, switchingTiming）を参照する共通取得口を定義する。
2. monthly/annual の `fetch*Summary` 呼び出しから固定値を撤去する。
3. 設定変更時に再取得される監視を追加する。
4. 画面上に現在の集計条件を表示する。

## Verification

- [ ] 設定変更後に月次画面の取得条件へ即時反映される
- [ ] 設定変更後に年次画面の取得条件へ即時反映される
- [ ] 画面の条件表示と実際のAPI引数が一致する

## Next Action

`settingsStore` と `monthlySummaryStore` / `annualSummaryStore` の接続点を先に確定する。

## Notes

- 実装変更に合わせて対象 SFC の `<spec lang="md">` 更新が必要な可能性あり。
