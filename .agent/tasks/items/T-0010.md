# [T-0010] 飲み物編集フォームのモーダル/ドロワー化

**作成日**: 2026-05-08  
**優先度**: High  
**種別**: enhancement  
**ステータス**: Backlog  
**担当バージョン**: -  
**関連 Issue**: -

## Goal

飲み物の編集を別ページ遷移ではなくインラインのモーダルまたはスライドアウトドロワーで完結させ、一覧のコンテキストを維持したまま編集できるようにする。

## Scope（対象ファイル・範囲）

- `app/pages/(protected)/drinks/index.vue`（編集ボタンのトリガー追加）
- `app/pages/(protected)/drinks/[id].vue`（廃止 or ドロワー呼び出しに変更）
- `app/pages/(protected)/drinks/new.vue`（同様にモーダル化を検討）
- `app/components/domain/drink/atoms/Edit.vue`（フォーム本体はそのまま再利用）
- `app/stores/pages/drinks/edit/`（initPage のルートパラム依存を解消）
- `app/stores/pages/drinks/new/`

## Out of Scope

- ラベル編集 (`/labels/:id`) の同様対応（別タスク化して構わない）
- Edit.vue のバリデーション追加（T-0014 で対応）

## Plan

1. `PageDrinkEditStore.initPage()` を route.params.id に依存しない形に変更し、drinkId を引数で受け取れるようにする
2. 一覧ページにドロワー/モーダル用の `isEditOpen` フラグと `editTargetDrinkId` を追加
3. 編集アイコンクリック時に遷移ではなくドロワーを開くよう変更
4. `DomainDrinkAtomsEdit` をドロワー内で利用
5. 保存/キャンセル後にドロワーを閉じ、一覧を再描画（ドリンクストアの更新で自動反映のはず）
6. `/drinks/:id` ページルートはドロワー未対応環境のフォールバックとして残す or 削除を検討

## Verification

- [ ] 一覧画面の編集アイコンをクリックするとドロワーが開き、フォームに既存データが入っている
- [ ] 保存するとドロワーが閉じて一覧が更新される
- [ ] キャンセルするとドロワーが閉じて一覧は変更されない
- [ ] モバイル幅でドロワーが正常に表示される

## Notes

- Bulma には Drawer コンポーネントがないため、`<div class="modal">` または CSS のサイドパネルで実装する
- `DomainDrinkAtomsEdit` は v-model を使っているため、ストア状態との接続方法を慎重に設計する
