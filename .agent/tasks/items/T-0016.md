## Goal

飲み物編集フォームの「量」フィールドラベルの可読性を改善し、カラーのHEXテキスト入力で無効文字列入力中にカラーピッカーがチラつく問題を解消する。

## Scope（対象ファイル・範囲）

- `app/components/domain/drink/atoms/Edit.vue`

## Out of Scope

- バリデーションエラーメッセージの追加（T-0014 で対応）

## Plan

1. **量フィールドのラベル改善**
   - 現状: `{{ t(LOCALE_DRINKS_AMOUNT) }} {{ t(LOCALE_DRINKS_STANDARD_AMOUNT) }} {{ selectedLabel?.standard_amount }}` を1つの `<label>` に連結
   - 変更後: `<label>` には `{{ t(LOCALE_DRINKS_AMOUNT) }}` のみ、標準量は `<p class="help">{{ t(LOCALE_DRINKS_STANDARD_AMOUNT) }}: {{ selectedLabel?.standard_amount }}</p>` として別行に表示

2. **カラーHEX入力のdebounce**
   - HEXテキスト入力には内部の `localColor` ref を用意し、debounce 300ms 後に有効な形式のみ `color` モデルに反映する
   - カラーピッカー（`<input type="color">`）は `color` モデルに直接 v-model して即時反映を維持する

## Verification

- [ ] 量フィールドのラベルが「1回あたりの量」のみで、標準量は補足テキストとして下に表示される
- [ ] HEXフィールドに `#FF` など不完全な文字列を入力してもカラーピッカーが即時変化しない
- [ ] 有効な HEX（例: `#FF5733`）を入力し終えると 300ms 後にカラーピッカーに反映される

## Notes

- debounce は `useDebounce` composable を利用するか、`setTimeout` + `clearTimeout` でシンプルに実装する
