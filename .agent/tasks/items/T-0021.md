# [T-0021] 飲み物フォームのラベル必須バリデーション追加

**作成日**: 2026-05-09  
**優先度**: High  
**種別**: enhancement  
**ステータス**: Backlog  
**担当バージョン**: -  
**関連 Issue**: -

## Goal

飲み物の新規作成・編集フォームでラベル未選択のまま保存できないようにする。ラベルなし飲み物を許容しないことで、T-0022（ラベル画面統合）の前提条件を満たす。

## Scope（対象ファイル・範囲）

- `app/components/domain/drink/atoms/Edit.vue`（バリデーション追加・select のプレースホルダー化）
- `app/utils/locales.ts`（`LOCALE_DRINKS_VALIDATION_LABEL_REQUIRED` 定数追加）
- `i18n/locales/ja.yaml`（翻訳追加）
- `i18n/locales/en.yaml`（翻訳追加）

## Out of Scope

- DB 制約変更（NOT NULL 化）は別途マイグレーションで対応する場合は別タスク
- 既存のラベルなし飲み物のマイグレーション処理

## Plan

1. **`locales.ts`** に `LOCALE_DRINKS_VALIDATION_LABEL_REQUIRED` 定数を追加
2. **`ja.yaml` / `en.yaml`** に翻訳を追加
   - ja: `'ラベルを選択してください'`
   - en: `'Please select a label'`
3. **`Edit.vue`** の変更
   - import に `LOCALE_DRINKS_VALIDATION_LABEL_REQUIRED` を追加
   - `errors` computed に `drinkLabel: !drinkLabelId.value ? t(LOCALE_DRINKS_VALIDATION_LABEL_REQUIRED) : null` を追加
   - ラベル select の空 option を `disabled` プレースホルダーに変更（`value=""` `disabled` `selected="!drinkLabelId"`）
   - select wrapper に `:class="{ 'is-danger': submitted && errors.drinkLabel }"` を追加
   - ラベルフィールド下部にエラーメッセージ表示 `<p v-if="submitted && errors.drinkLabel" class="help is-danger">` を追加

## Verification

- [ ] ラベル未選択で保存ボタンを押すとエラーメッセージが表示される
- [ ] ラベルを選択するとエラーが消える
- [ ] ラベルを選択済みの場合は保存が通常通り動作する
- [ ] 編集画面（既存ラベルあり）でも正常に動作する

## Notes

- 既存のラベルなし飲み物がある場合、編集画面で再保存しようとするとエラーが出る。既存データの扱いは運用で対応（ラベルを設定してから保存）
- T-0022（画面統合・名称変更）の前提タスク
