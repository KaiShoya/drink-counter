# [T-0022] 飲み物ラベル画面を「飲み物」に統合・名称変更

**ID**: T-0022  
**作成日**: 2026-05-09  
**優先度**: Medium  
**種別**: enhancement  
**ステータス**: Backlog  
**担当バージョン**: -  
**関連 Issue**: -

---

## Goal

T-0020 でラベル詳細画面に飲み物一覧・並び替えが追加されたことで、独立した「飲み物一覧」画面（`/drinks`）の主要ユースケースがラベル詳細画面に代替された。「飲み物ラベル」画面を「飲み物」に名称変更・統合し、ナビゲーション構造を簡略化する。

## Scope（対象ファイル・範囲）

### ルーティング
- `app/pages/(protected)/labels/` → `app/pages/(protected)/drinks/` にディレクトリ名変更
- 旧 `app/pages/(protected)/drinks/` の index/new/[id] の扱いを決定（後述）

### ナビ・i18n
- ナビバーの「飲み物ラベル」エントリを「飲み物」に変更（`routes.drinks` を再利用 or `routes.labels` を更新）
- `i18n/locales/ja.yaml` / `en.yaml` のルートキー整理

### リダイレクト
- `/labels` → `/drinks`（Nuxt のルートエイリアスまたは middleware で 301 相当の処理）
- `/labels/:id` → `/drinks/:id`

### 旧 `/drinks` ページの処理
- `drinks/index.vue`（全件一覧）: **廃止** または「ラベル未設定の飲み物」専用ページとして保持するかを決定
- `drinks/new.vue`: ラベル詳細画面の「このラベルに追加」リンク経由で引き続き利用するため **保持**
- `drinks/[id].vue`（編集）: 引き続き利用するため **保持**

## Out of Scope

- T-0010（飲み物編集のモーダル化）との統合（独立して実施）
- DB レベルでの `drink_label_id NOT NULL` 制約変更

## Dependencies

- **T-0021 完了が前提**（ラベルなし飲み物を許容しないバリデーションが必要）

## Plan

1. ナビバーのリンク先・ラベルを `/labels` → `/drinks`（新）に変更
2. `app/pages/(protected)/labels/` を `app/pages/(protected)/drinks/` にリネーム（`mv` コマンド）
3. 旧 `drinks/index.vue`（全件一覧）の廃止方針を確認・対応
   - 廃止する場合: ファイル削除、旧 URL へのアクセスはトップや新 `/drinks` にリダイレクト
4. `drinks/new.vue` と `drinks/[id].vue` を適切なパスに移動（ネスト構造を考慮）
5. i18n の `routes.labels` キーを `routes.drinks` に統合 or 削除（競合解消）
6. 全ファイル内の `/labels` パス参照を検索・一括置換
7. `localePath('/labels')` → `localePath('/drinks')` の置換

## Verification

- [ ] ナビバーの「飲み物」をクリックするとラベル一覧（旧: ラベル画面）が表示される
- [ ] `/labels` にアクセスすると `/drinks` にリダイレクトされる
- [ ] `/labels/:id` にアクセスすると `/drinks/:id` にリダイレクトされる
- [ ] ラベル作成・編集・削除が正常に動作する
- [ ] 飲み物新規追加・編集が正常に動作する
- [ ] ナビバーに「飲み物ラベル」の重複エントリが残らない

## Notes

- `localePath` を多用しているため、パス文字列の変更漏れに注意。`grep -r '/labels'` で網羅的に確認する
- Vercel 等の本番環境でも旧 URL からのリダイレクトが機能するか確認が必要
- ページのタイトル (`useSeoMeta`) も合わせて更新すること
