# [T-0012] ラベル選択時の量自動上書き通知の追加

**作成日**: 2026-05-08

**種別**: enhancement  
**ステータス**: Backlog  
**担当バージョン**: -  
**関連 Issue**: -

---

## Goal

飲み物編集フォームでラベルを変更したとき `amount` がサイレントに上書きされる問題を解消し、ユーザーが意図的に標準量を反映できるようにする。

## Scope（対象ファイル・範囲）

- `app/components/domain/drink/atoms/Edit.vue`（`changeDrinkLabelId` 関数の変更）

## Out of Scope

- ラベル編集フォーム（standard_amount 自体の変更であり動作が異なる）

## Plan

1. `changeDrinkLabelId` 内で `amount.value = selectedLabel.value?.standard_amount` を即時実行するのをやめる
2. ラベル変更時、`amount` フィールドの下に「標準量 ◯ml を反映しますか？」のインラインヒント（Bulma の `help` テキスト）とボタン「反映する」を表示する
3. 「反映する」ボタン押下時のみ `amount.value` を更新し、ヒントを非表示にする
4. 別のラベルを再選択した場合はヒントをリセットする

## Verification

- [ ] ラベルを変更しても amount フィールドの値は自動で変わらない
- [ ] 標準量が設定されているラベルを選択するとインラインヒントが表示される
- [ ] 「反映する」を押すと amount が標準量で上書きされ、ヒントが消える
- [ ] ラベルを「選択なし」に戻したらヒントが消える

## Notes

- 標準量が null / undefined のラベルを選択した場合はヒントを表示しない
